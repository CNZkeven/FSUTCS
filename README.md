# 燃油供应单元温度控制系统课程设计说明
# Course-Design-for-the-Fuel-Supply-Unit-Temperature-Control-System

## 1. 课题背景与设计目标

本项目面向“燃油供应单元温度控制系统设计”课程设计要求，使用 STM32F407 作为下位机控制核心，使用 PT100 温度检测信号作为反馈量，使用 DAC 电压输出驱动外部可控硅调压执行机构，并通过 Python 上位机实现参数整定、实时监控、曲线显示和数据记录。

课程设计的核心目标不是单纯“把温度读出来”，而是完成一个完整的闭环温控系统：

- 能够采集温度并完成温度换算与标定
- 能够根据设定值执行 PID 控制
- 能够输出控制量驱动加热执行机构
- 能够完成开始、停止、超温保护和报警
- 能够通过串口与上位机双向通信
- 能够在 LCD 和上位机界面同时显示系统运行状态

## 3. 系统总体架构

整个系统由“温度采集、闭环控制、执行输出、人机交互、串口通信”五部分组成。

```text
          +-------------------------------+
          |         Python 上位机         |
          | 参数下发 / 曲线显示 / 记录数据 |
          +---------------+---------------+
                          |
                    USART1 串口
                    115200, 8N1
                          |
          +---------------v---------------+
          |         STM32F407 下位机      |
          |  温度采样 / PID 控制 / 状态机  |
          +-------+---------------+-------+
                  |               |
             ADC 采样             | DAC 输出
                  |               |
          +-------v----+     +----v-------+
          | PT100 温度 |     | 可控硅调压 |
          | 传感器信号 |     | 加热执行端 |
          +------------+     +------------+
                  |
             被控对象温度

另外，STM32 本地还通过 LCD 显示运行状态与控制参数。
```

## 4. 硬件与软件设计

### 4.1 下位机设计

下位机位于 `code/` 目录，核心任务如下：

- 通过 `ADC1` 采集 PT100 调理后的模拟量
- 将 ADC 原始值换算为温度值
- 根据设定温度与当前温度计算 PID 输出
- 将控制输出映射为 DAC 电压，用于控制可控硅调压
- 在 LCD 上显示当前温度、设定温度、ADC/DAC、PID 参数和报警状态
- 通过 USART1 接收上位机指令，并周期性回传系统状态

### 4.2 上位机设计

上位机位于 `app/` 目录，主要功能如下：

- 配置串口并与 STM32 建立连接
- 下发设定温度、PID 参数和超温阈值
- 接收 MCU 回传的遥测数据
- 实时显示当前温度、设定值、输出电压和 DAC 数值
- 绘制温度曲线、设定曲线和输出曲线
- 将调试过程保存为 `record.csv`

## 5. 控制流程设计

系统采用典型闭环温度控制结构，主流程如下：

1. STM32 完成时钟、串口、LCD、ADC、DAC 等外设初始化。
2. 主循环内对 `ADC_Channel_6` 连续采样 20 次求平均，降低噪声影响。
3. 将平均 ADC 值通过 `adc2temp()` 转换为当前温度。
4. 判断当前温度是否超过 `over_limit` 超温阈值，若超过则立即报警并停止加热。
5. 若系统处于运行状态，则执行 `pid_realize()` 计算控制量。
6. 将控制量映射为 DAC 码值，进一步转换为实际控制电压输出给执行器。
7. 刷新 LCD 界面，并通过串口输出一帧运行数据供上位机显示和记录。
8. 检查串口接收缓冲区中是否已有完整命令，如有则解析并更新控制参数。

该流程的特点是：

- 采样、控制、显示、通信都集中在同一主循环中，结构清晰，便于课设讲解。
- 串口通信不直接参与实时 PID 运算，只承担参数整定和状态监视任务，避免通信异常直接破坏控制主链路。
- 超温保护优先级高于正常控温逻辑，保证系统安全。

## 6. PID 控制策略说明

本系统使用位置式 PID 控制思想。核心计算位于 `code/USER/main.c` 中的 `pid_realize()`。

控制逻辑为：

```text
误差 e(k) = 目标温度 - 当前温度
积分项 = 积分项 + e(k) * dt
微分项 = (e(k) - e(k-1)) / dt
输出 u(k) = P * e(k) + I * 积分项 + D * 微分项
```

程序中还做了两个工程化处理：

- 对输出做限幅，防止控制量超过执行机构允许范围
- 对负控制量钳位到 0，因为当前执行端是加热装置，控制通常只需要单向输出

这种实现方式适合课程设计，因为它计算直观，便于说明比例、积分、微分三项各自的作用，也方便通过上位机在线调整参数。

## 7. 串口通信设计

串口通信是本项目最重要的“联机调试通道”，它把下位机控制闭环和上位机观测界面连接起来，也是老师最可能追问的部分。

### 7.1 通信参数

- 串口外设：`USART1`
- 波特率：`115200`
- 数据位：`8`
- 停止位：`1`
- 校验位：`None`
- 流控：`None`
- 指令结束符：`\r\n`

选择 `115200, 8N1` 的原因是：

- 通用 USB 转串口模块普遍支持，便于实验室连接
- 对 100ms 一帧的数据上报完全够用
- 配置简单，便于在上位机与 MCU 之间统一

### 7.2 下位机到上位机的数据上报

STM32 每个控制周期都会通过 `printf()` 输出一行 CSV 文本：

```text
TEMP,SET,P,I,D,DAC,VOLT,RUN,ALARM,LIMIT
```

示例：

```text
45.21,60.00,50.00,0.10,0.00,2140,1.233,1,0,75.00
```

各字段含义如下：

| 字段 | 含义 |
| --- | --- |
| `TEMP` | 当前温度 |
| `SET` | 当前设定温度 |
| `P/I/D` | 当前 PID 参数 |
| `DAC` | 当前 DAC 原始码值 |
| `VOLT` | 按输出级标定后的控制电压 |
| `RUN` | 运行状态，`1` 为运行，`0` 为停止 |
| `ALARM` | 报警状态，`1` 为报警 |
| `LIMIT` | 当前超温阈值 |

这样设计的优点是：

- 上位机无需读取寄存器，只需按文本协议解析即可
- 一行数据包含控制分析所需的主要量，方便曲线显示与记录
- CSV 格式便于直接保存到文件并做后续实验分析

### 7.3 上位机到下位机的控制指令

上位机向 MCU 发送的命令如下，均以 `\r\n` 结尾：

```text
START
STOP
TEMP:xx.xx
P:xx.xx
I:xx.xx
D:xx.xx
LIMIT:xx.xx
GET
```

其中：

- `START`：启动闭环加热
- `STOP`：停止控制输出
- `TEMP`：设置目标温度
- `P/I/D`：设置 PID 参数
- `LIMIT`：设置超温保护阈值
- `GET`：请求 MCU 回显当前参数

### 7.4 下位机串口接收实现

下位机串口接收位于 `code/SYSTEM/usart/usart.c`，采用“串口中断 + 缓冲区 + 主循环解析”的结构。

实现思路如下：

1. `USART1_IRQHandler()` 在接收到每个字节时进入中断。
2. 程序把普通字符依次写入 `USART_RX_BUF`。
3. 当接收到 `0x0d`（`\r`）时，置位“收到回车”标志。
4. 若下一个字符是 `0x0a`（`\n`），则置位“接收完成”标志。
5. 主循环中的 `usart_process()` 检测到完整一帧后，再将缓冲区内容转为字符串并调用 `parse_command()`。

这种设计的优点是：

- 中断中只做轻量级收字节，不在中断里解析复杂命令
- 主循环统一处理业务逻辑，结构清楚
- 使用 `\r\n` 作为结束符，可以避免半包数据被错误解析

### 7.5 上位机串口处理实现

上位机串口管理位于 `app/core/serial_manager.py`，基于 `QSerialPort` 实现。

接收流程如下：

1. 串口有新数据时触发 `readyRead` 信号。
2. `_on_ready_read()` 把字节流追加到 `_buffer` 缓冲区。
3. 以 `\n` 作为分帧依据，将完整行取出并解码为字符串。
4. `MainWindow._on_line()` 判断当前行是否为 10 字段遥测帧。
5. 若是遥测帧，则解析并刷新界面、曲线和数据记录；否则作为普通日志显示。

发送流程如下：

1. `build_command()` 统一构造命令文本，并自动补上 `\r\n`。
2. 单条命令可直接通过 `send()` 发送。
3. 参数批量下发时，调用 `send_batch()` 逐条发送，并设置 `120ms` 间隔。

这里专门加发送间隔的原因是：

- MCU 当前接收设计是一帧一帧处理的
- 主循环本身就以约 `100ms` 为周期刷新一次
- 若连续无间隔发送多条命令，可能出现多条命令短时间堆积，影响演示稳定性

因此，上位机采用“批量命令分时发送”的方式，更适合课程设计演示和现场调参。

### 7.6 串口通信在整个系统中的作用

从系统角度看，串口通信并不是控制回路本体，而是“控制参数配置 + 运行状态观测 + 实验数据采集”的桥梁：

- 没有串口时，系统仍可在本地 LCD 上独立运行
- 有了串口后，可在线修改设定值和 PID 参数
- 上位机能把调试过程转化为可记录、可分析、可展示的实验数据

这也是本项目课设价值的重要体现，因为它把“嵌入式控制”与“上位机监控”真正结合了起来。

## 10. 运行方式

### 10.1 下位机构建

工程已提供 CMake 构建脚本，构建方式如下：

```bash
cmake -S code -B code/build
cmake --build code/build
```

构建完成后可生成：

- `code/build/ITEMP.elf`
- `code/build/ITEMP.hex`
- `code/build/ITEMP.bin`

### 10.2 上位机运行

先安装依赖：

```bash
python -m pip install -r requirements.txt
```

然后在项目根目录运行：

```bash
python app/main.py
```

### 10.3 串口连接参数

- 串口：连接到开发板对应的 USB 转串口端口
- 波特率：`115200`
- 数据格式：`8N1`

## 11. 实验与调试说明

建议按照以下顺序演示：

1. 上电后观察 LCD，确认当前温度、设定值、PID 参数显示正常。
2. 打开上位机并连接串口。
3. 先下发 `TEMP`、`P`、`I`、`D`、`LIMIT` 参数。
4. 点击 `START`，观察温度曲线逐渐趋近设定值。
5. 调整 PID 参数，比较超调量和稳定速度变化。
6. 降低 `LIMIT` 阈值，验证超温报警和停机功能。
7. 点击“开始记录”，在实验结束后保存 `record.csv`，用于报告中的曲线分析。
